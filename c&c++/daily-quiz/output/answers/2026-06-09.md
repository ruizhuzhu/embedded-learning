## 第 1 题答案 — 选择题 — ★☆☆

**正确答案：C**

**详细解析：**

`arr` 是指向数组首元素的指针。`arr + 2` 指向第三个元素（索引 2），`*(arr + 2)` 等价于 `arr[2]`，值为 3。数组下标 `arr[i]` 在 C 语言中本质上就是 `*(arr + i)` 的语法糖。

## 第 2 题答案 — 选择题 — ★★☆

**正确答案：B**

**详细解析：**

SPI 四种模式快速记忆：
- **CPOL** = Clock Polarity：空闲时电平（0=低，1=高）
- **CPHA** = Clock Phase：采样边沿（0=第一个边沿，1=第二个边沿）

| 模式 | CPOL | CPHA | 空闲电平 | 采样边沿 |
|------|------|------|----------|----------|
| 0 | 0 | 0 | 低 | 第一边沿=上升沿 |
| 1 | 0 | 1 | 低 | 第二边沿=下降沿 |
| 2 | 1 | 0 | 高 | 第一边沿=下降沿 |
| 3 | 1 | 1 | 高 | 第二边沿=上升沿 |

设备手册通常标明采用的模式，主从必须匹配。Mode 0 和 Mode 3 是最常用的。

## 第 3 题答案 — 代码分析 — ★★★

**问题分析与修复：**

**I2C NACK 的常见原因**：
1. **从机地址错误**：7 位地址移位问题（有些库需要提供左移 1 位后的值，有些是原始 7 位地址）
2. **从机不存在/未上电**：目标设备未焊接、供电问题、复位状态
3. **从机忙**：从机正在处理上次命令（如 EEPROM 内部写周期），暂时不应答
4. **总线电平问题**：上拉电阻过大（>10kΩ）导致上升沿太慢，波形不满足时序要求
5. **时钟频率过高**：超过了从机能支持的最大速率
6. **时序违例**：启动条件/停止条件的建立和保持时间不满足
7. **数据格式错误**：发送的命令/寄存器地址从机无法识别

**排查方法**：
1. 用逻辑分析仪或示波器抓取 SDA/SCL 波形，确认地址、ACK 位、时序
2. 用 I2C 扫描器扫描全部地址，确认从机是否在线
3. 降低时钟频率测试
4. 检查上拉电阻值（典型 4.7kΩ for 3.3V、2.2kΩ for 1.8V）
5. 检查设备 RESET 引脚状态

**详细解析：**

I2C 调试是嵌入式工程师最常见的实战场景之一，考察排查问题的系统性思维。

## 第 4 题答案 — 简答题 — ★★★

**参考答案：**

**HardFault 排查步骤**：

1. **确认故障类型**：读取 CFSR（地址 0xE000ED28）
   - `CFSR[7:0]` = MFSR（MemManage Fault）：非对齐访问、MPU 违规、访问无效地址
   - `CFSR[15:8]` = BFSR（Bus Fault）：总线错误（如访问未启用的外设）
   - `CFSR[31:16]` = UFSR（Usage Fault）：未定义指令、除零、非对齐（关闭时）、协处理器访问

2. **获取故障地址**：
   - MMFAR（0xE000ED34）：触发 MemManage Fault 的地址
   - BFAR（0xE000ED38）：触发 Bus Fault 的地址

3. **从栈帧获取故障时的 PC 和 LR**：
   ```c
   void HardFault_Handler_C(uint32_t *hardfault_args) {
       uint32_t stacked_r0  = hardfault_args[0];
       uint32_t stacked_r1  = hardfault_args[1];
       uint32_t stacked_r2  = hardfault_args[2];
       uint32_t stacked_r3  = hardfault_args[3];
       uint32_t stacked_r12 = hardfault_args[4];
       uint32_t stacked_lr  = hardfault_args[5];  // 故障时的返回地址
       uint32_t stacked_pc  = hardfault_args[6];  // 故障时的程序计数器
       uint32_t stacked_psr = hardfault_args[7];
       
       printf("Fault at PC: 0x%08X, LR: 0x%08X\n", stacked_pc, stacked_lr);
       // 用 addr2line 或反汇编窗口定位源代码行
   }
   
   __attribute__((naked))
   void HardFault_Handler(void) {
       __asm volatile(
           "TST LR, #4\n"        // 检查 EXC_RETURN bit2 确定 MSP/PSP
           "ITE EQ\n"
           "MRSEQ R0, MSP\n"
           "MRSNE R0, PSP\n"
           "B HardFault_Handler_C\n"
       );
   }
   ```

4. **常见故障原因检查清单**：
   - 栈溢出（数组越界、递归过深）→ 检查栈 watermark
   - 访问 NULL / 野指针 → PC 在异常地址
   - 外设时钟未使能就访问其寄存器 → BFAR 指向外设地址
   - 非对齐访问 → CFSR 的 MFSR 位被置

**详细解析：**

HardFault 调试是嵌入式开发者的必备技能，面试常考实际调试思路。
