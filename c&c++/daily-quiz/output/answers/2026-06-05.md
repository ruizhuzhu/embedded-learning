**正确答案：B**

**详细解析：**

union 的所有成员共享同一块内存（起始地址相同）。在小端系统中，最低字节（0x78）存储在最低地址，所以 `data.bytes[0]` 访问的是最低地址的字节 = 0x78。大端系统则输出 0x12。Union 常用于协议解析（将字节数组和结构体共用）、大小端判断等场景。

## 第 3 题答案 — 选择题 — ★☆☆

**正确答案：B**

**详细解析：**

ISR 设计的黄金法则：Keep it Short and Simple（KISS）。
- ISR 执行时会屏蔽同级或更低优先级的中断，过长会导致其他中断丢失
- 一般来说 ISR 只做必要的事：清除中断标志、读取/写入关键数据、发送信号量通知任务
- 复杂处理放到任务或下半部（Bottom Half）中完成
- 不应在 ISR 中调用可能阻塞的函数（如 printf、vTaskDelay）、获取互斥锁（应使用 FromISR 系列函数）

## 第 4 题答案 — 编程题 — ★★★

**参考实现：**

```c
// 全局变量
static uint32_t last_capture = 0;
static uint32_t overflow_count = 0;
static float measured_freq = 0;

void TIM2_Config(void) {
    // 1. 使能时钟
    RCC->APB1ENR |= RCC_APB1ENR_TIM2EN;
    
    // 2. 配置 GPIO 为复用功能（AF）
    // GPIOA->MODER, AFR[0] 配置 PA0 为 AF1 (TIM2_CH1)
    
    // 3. 定时器基本配置
    TIM2->PSC = 71;        // 72MHz / 72 = 1MHz → 1us 分辨率
    TIM2->ARR = 0xFFFFFFFF; // 最大计数范围（32位）
    
    // 4. 输入捕获配置
    TIM2->CCMR1 |= (0x01 << 0);   // CH1 输入，映射到 TI1
    TIM2->CCER  |= TIM_CCER_CC1E;  // 使能 CH1 捕获
    TIM2->CCER  |= (0x01 << 1);    // 上升沿触发
    TIM2->DIER  |= TIM_DIER_CC1IE | TIM_DIER_UIE; // 捕获中断 + 溢出中断
    TIM2->CR1   |= TIM_CR1_CEN;    // 使能定时器
    
    // 5. NVIC 配置
    NVIC_EnableIRQ(TIM2_IRQn);
}

void TIM2_IRQHandler(void) {
    // 溢出处理
    if (TIM2->SR & TIM_SR_UIF) {
        TIM2->SR = ~TIM_SR_UIF;
        overflow_count++;
    }
    
    // 捕获处理
    if (TIM2->SR & TIM_SR_CC1IF) {
        uint32_t current = TIM2->CCR1;
        
        // 计算两个上升沿之间的时间（处理溢出）
        uint64_t total_ticks = (uint64_t)overflow_count * 0x100000000ULL 
                             + current - last_capture;
        
        measured_freq = 1000000.0f / total_ticks;  // 1us 分辨率 → Hz
        
        last_capture = current;
        overflow_count = 0;
        
        TIM2->SR = ~TIM_SR_CC1IF;
    }
}
```

**详细解析：**

输入捕获测频的关键点：
1. 使用 32 位定时器 + 溢出计数防止低频率测量溢出
2. 分辨率选择：PSC 越大，分辨率越低但最大测量时间越长
3. 信号调理：外部信号可能需要施密特触发器整形
4. 如果频率很高，改用 PWM 输入模式（双捕获，一个周期内获得周期和占空比）
