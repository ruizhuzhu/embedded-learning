## 第 2 题答案 — 编程题 — ★★☆

**参考实现：**

```c
#define BUF_SIZE 64
static RingBuf rb;
static SemaphoreHandle_t xDataReadySem;  // 数据就绪信号量
static SemaphoreHandle_t xBufMutex;      // 缓冲互斥锁

void vADCTask(void *pvParameters) {
    for (;;) {
        uint16_t adc_val = ADC_Read();
        
        if (xSemaphoreTake(xBufMutex, portMAX_DELAY) == pdTRUE) {
            if (!rb_is_full(&rb)) {
                rb_push(&rb, adc_val);
                xSemaphoreGive(xDataReadySem);  // 通知有新数据
            }
            xSemaphoreGive(xBufMutex);
        }
        vTaskDelay(pdMS_TO_TICKS(10));
    }
}

void vProcessTask(void *pvParameters) {
    for (;;) {
        // 等待数据就绪
        if (xSemaphoreTake(xDataReadySem, portMAX_DELAY) == pdTRUE) {
            xSemaphoreTake(xBufMutex, portMAX_DELAY);
            uint16_t data = rb_pop(&rb);
            xSemaphoreGive(xBufMutex);
            
            ProcessData(data);
        }
    }
}
```

**详细解析：**

关键设计点：
1. 使用信号量通知有数据就绪（而非忙等轮询）
2. 互斥锁保护环形缓冲区的并发访问
3. 生产者检查缓冲区满，而非覆盖数据
4. 使用 `portMAX_DELAY` 确保可靠等待
5. 生产者的 `vTaskDelay` 控制采样频率

## 第 3 题答案 — 编程题 — ★★☆

**参考实现：**

```c
#define FILTER_WINDOW 8

typedef struct {
    uint16_t buffer[FILTER_WINDOW];
    uint32_t sum;
    uint8_t  index;
    uint8_t  count;  // 已填充的样本数
} MovingAvgFilter;

static MovingAvgFilter adc_filt = {0};

uint16_t adc_filter(uint16_t new_sample) {
    // 如果窗口已满，减去即将被覆盖的旧值
    if (adc_filt.count >= FILTER_WINDOW) {
        adc_filt.sum -= adc_filt.buffer[adc_filt.index];
    } else {
        adc_filt.count++;
    }
    
    // 存入新值，更新总和
    adc_filt.buffer[adc_filt.index] = new_sample;
    adc_filt.sum += new_sample;
    
    // 移动索引
    adc_filt.index = (adc_filt.index + 1) % FILTER_WINDOW;
    
    // 返回平均值
    return (uint16_t)(adc_filt.sum / adc_filt.count);
}

// 指数移动平均（EMA）版本 —— 更省内存
uint16_t adc_filter_ema(uint16_t new_sample) {
    static uint16_t filtered = 0;
    static bool first = true;
    
    if (first) {
        filtered = new_sample;
        first = false;
    } else {
        // alpha = 0.125 (1/8), 新的占 1/8, 旧的占 7/8
        filtered = filtered + ((int16_t)(new_sample - filtered) >> 3);
    }
    return filtered;
}
```

**详细解析：**

滑动窗口优化要点：
1. 维护一个 sum，避免每次重新计算 8 个数的和（从 O(N) 降到 O(1)）
2. 用 count 处理初始填充阶段（不足窗口大小时只用已有数据）
3. 环形缓冲区 index 处理覆盖逻辑
4. EMA 版本仅需 2 字节 RAM，适合极端资源受限场景

权衡：移动平均对突变响应慢但平滑效果好；EMA 省内存、响应更快但滤波效果稍弱。
