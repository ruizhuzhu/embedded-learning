## 第 1 题答案 — 选择题 — ★★☆

**正确答案：B**

**详细解析：**

FreeRTOS 抢占式调度中，任务切换触发条件：
1. 更高优先级任务进入就绪态（中断中释放信号量、发送消息队列等）
2. 当前运行任务进入阻塞/挂起态（等待信号量、延迟等）
3. 时间片轮转：同优先级任务的时间片到期（如果 configUSE_TIME_SLICING = 1）
4. 中断退出时（如果 configUSE_PREEMPTION = 1），检查是否需要切换到更高优先级任务

理解这些对于设计实时系统至关重要。

## 第 2 题答案 — 选择题 — ★★☆

**正确答案：B**

**详细解析：**

**Bootloader 职责**：
1. 初始化基本硬件（时钟、Flash 控制器、通信接口）
2. 检查升级标志，决定进入升级模式还是直接引导
3. 通过 UART/SPI/USB/CAN 接收新固件并写入 Flash
4. 验证固件完整性（CRC/签名校验）
5. 跳转到应用程序

**双区备份优势**：
- Flash 分为 A 区和 B 区，旧固件在 A 区运行，新固件下载到 B 区
- 升级失败（断电、校验失败）→ 回滚到 A 区，设备不会变砖
- 支持 A/B 无缝切换（swap 后重启）
- 缺点是需要双倍 Flash 空间

## 第 3 题答案 — 选择题 — ★☆☆

**正确答案：C**

**详细解析：**

- **.text**: 代码/指令
- **.data**: 已初始化的全局/静态变量
- **.bss**: 未初始化（或初始化为 0）的全局/静态变量，启动时清零
- **.stack**: 局部变量、函数调用返回地址
- **.heap**: 动态分配（malloc）的内存

嵌入式启动代码中通常需要清零 .bss 段并初始化 .data 段（从 ROM 复制初始值）。

## 第 4 题答案 — 编程题 — ★★☆

**参考实现：**

```c
typedef enum { STATE_IDLE, STATE_SHORT_PRESS, STATE_LONG_PRESS } State;
typedef enum { EVENT_PRESS_DOWN, EVENT_PRESS_UP, EVENT_TIMEOUT_1S } Event;

typedef State (*StateHandler)(Event);

// 状态处理函数声明
State idle_handler(Event e);
State short_handler(Event e);
State long_handler(Event e);

// 状态转移表
typedef struct {
    State state;
    StateHandler handler;
    void (*on_enter)(void);
    void (*on_exit)(void);
} StateEntry;

static uint32_t press_start_time = 0;

StateEntry state_table[] = {
    {STATE_IDLE,        idle_handler,  NULL, NULL},
    {STATE_SHORT_PRESS, short_handler, NULL, NULL},
    {STATE_LONG_PRESS,  long_handler, NULL, NULL},
};

static State current_state = STATE_IDLE;

void fsm_dispatch(Event e) {
    State next = state_table[current_state].handler(e);
    if (next != current_state) {
        if (state_table[current_state].on_exit)
            state_table[current_state].on_exit();
        if (state_table[next].on_enter)
            state_table[next].on_enter();
        current_state = next;
    }
}

State idle_handler(Event e) {
    switch (e) {
        case EVENT_PRESS_DOWN:
            press_start_time = get_tick();
            return STATE_SHORT_PRESS;
        default:
            return STATE_IDLE;
    }
}

State short_handler(Event e) {
    switch (e) {
        case EVENT_PRESS_UP:
            toggle_led();   // 短按：切换 LED
            return STATE_IDLE;
        case EVENT_TIMEOUT_1S:
            enter_setting(); // 长按：进入设置
            return STATE_LONG_PRESS;
        default:
            return STATE_SHORT_PRESS;
    }
}

State long_handler(Event e) {
    switch (e) {
        case EVENT_PRESS_UP:
            return STATE_IDLE;
        default:
            return STATE_LONG_PRESS;
    }
}
```

**详细解析：**

状态机设计要点：
1. 转移表让状态机结构清晰，新增状态只需添加表项
2. on_enter/on_exit 动作确保状态切换时的副作用可控
3. 事件驱动：主循环或中断中调用 `fsm_dispatch(EVENT_xxx)`
4. 对于复杂状态机，可引入层次状态机（HSM）或使用状态机生成工具
