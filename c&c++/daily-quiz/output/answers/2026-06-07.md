## 第 1 题答案 — 选择题 — ★☆☆

**正确答案：B**

**详细解析：**

开漏输出的特点：
- 输出寄存器为 0 时：引脚拉到 GND（低电平）
- 输出寄存器为 1 时：引脚高阻态（浮空），由外部上拉电阻决定高电平
- 多个开漏输出可以连接在一起，任何一个拉低都会使总线变低（线与逻辑）

典型应用：
1. **I2C 总线**：SDA 和 SCL 都需要开漏输出 + 外部上拉
2. **1-Wire 总线**：如 DS18B20 温度传感器
3. **电平转换**：5V 容忍引脚接 3.3V MCU，通过上拉到 5V 实现
4. **LED 多路复用**：多个 LED 共阳连接

## 第 2 题答案 — 简答题 — ★☆☆

**参考答案：**

**帧错误（Framing Error）**：
- 接收端在 STOP 位的预期位置检测到低电平而非高电平
- 原因：波特率不匹配、时钟偏差过大、线路噪声干扰
- 产生条件：通常双方的波特率误差总和超过约 5% 时开始出现

**溢出错误（Overrun Error）**：
- 接收移位寄存器收到新数据时，接收数据寄存器（RDR）中的数据还未被 CPU 读取
- 原因：中断延迟过大 / 中断被长时间屏蔽 / 未使用 FIFO
- 结果：新数据覆盖了旧数据

**波特率误差的影响**：
- UART 在每帧中间（bit 中心）采样，16x 过采样
- 累积误差：10 位帧（1 start + 8 data + 1 stop），最后一位（stop）采样偏差最大
- 5% 误差意味着 stop 位采样点时偏差约半个位周期，容易出错
- 工程建议：波特率误差控制在 2% 以内
- 使用外部晶振（而非内部 RC 振荡器）以获得精确波特率

**波特率计算公式**：BRR = PCLK / (16 × BaudRate)，需验证实际值和目标值的偏差。

## 第 3 题答案 — 简答题 — ★☆☆

**参考答案：**

**原因**：
1. 避免魔法数字，提高代码可读性
2. 数组大小变化时无需手动修改所有相关代码
3. 减少维护错误

**安全宏（两种常见写法）**：
```c
// 方式一：基本版本
#define ARRAY_SIZE(x) (sizeof(x) / sizeof((x)[0]))

// 方式二：编译期检查版本（如果传入指针而非数组，编译报错）
#define ARRAY_SIZE(x) (sizeof(x) / sizeof((x)[0]) \
    + (int)(sizeof(struct { int:-!!((void*)(x) != (void*)&(x)); })))

// C11/C++11 可用静态断言版本
#define ARRAY_SIZE(x) \
    (_Static_assert(!__builtin_types_compatible_p(typeof(x), typeof(&(x)[0])), \
                    "Not an array"), \
     sizeof(x) / sizeof((x)[0]))
```

**详细解析：**

这个宏在嵌入式代码中出镜率极高。高级版本的类型检查非常重要——如果错误地将指针传入，基本版会静默返回错误结果（指针大小 / 元素大小），导致隐蔽 bug。

## 第 4 题答案 — 编程题 — ★★★

**参考实现：**

```c
typedef enum {
    STATE_WAIT_HEAD1,
    STATE_WAIT_HEAD2,
    STATE_LENGTH,
    STATE_PAYLOAD,
    STATE_CHECKSUM,
    STATE_ESCAPE
} ParserState;

typedef struct {
    ParserState state;
    uint8_t buf[256];
    uint8_t length;
    uint8_t index;
    uint8_t checksum;
    bool    escape_next;
} ProtoParser;

void parser_init(ProtoParser *p) {
    p->state = STATE_WAIT_HEAD1;
    p->index = 0;
    p->escape_next = false;
}

typedef enum {
    PARSE_INCOMPLETE = 0,
    PARSE_OK = 1,
    PARSE_CHECKSUM_ERR = -1,
    PARSE_LENGTH_ERR = -2
} ParseResult;

ParseResult parser_feed(ProtoParser *p, uint8_t byte, 
                         uint8_t **payload, uint8_t *payload_len) {
    // 转义处理（最高优先级）
    if (byte == 0xAA && !p->escape_next && p->state != STATE_WAIT_HEAD1) {
        p->escape_next = true;
        return PARSE_INCOMPLETE;
    }
    
    if (p->escape_next) {
        p->escape_next = false;
        if (byte != 0x00 && byte != 0xAA) {
            goto reset;  // 非法转义序列，重置
        }
        // 否则 byte 继续正常处理（0xAA 或 0x00）
    }
    
    switch (p->state) {
    case STATE_WAIT_HEAD1:
        if (byte == 0xAA) {
            p->state = STATE_WAIT_HEAD2;
        }
        return PARSE_INCOMPLETE;
        
    case STATE_WAIT_HEAD2:
        if (byte == 0x55) {
            p->state = STATE_LENGTH;
        } else {
            p->state = STATE_WAIT_HEAD1;
        }
        return PARSE_INCOMPLETE;
        
    case STATE_LENGTH:
        p->length = byte;
        p->checksum = byte;  // 校验和包含长度字节
        p->index = 0;
        if (p->length > 255) goto reset;
        p->state = (p->length > 0) ? STATE_PAYLOAD : STATE_CHECKSUM;
        return PARSE_INCOMPLETE;
        
    case STATE_PAYLOAD:
        p->buf[p->index++] = byte;
        p->checksum ^= byte;
        if (p->index >= p->length) {
            p->state = STATE_CHECKSUM;
        }
        return PARSE_INCOMPLETE;
        
    case STATE_CHECKSUM:
        p->state = STATE_WAIT_HEAD1;
        if (byte == p->checksum) {
            *payload = p->buf;
            *payload_len = p->length;
            return PARSE_OK;
        }
        return PARSE_CHECKSUM_ERR;
    }
    
reset:
    p->state = STATE_WAIT_HEAD1;
    p->escape_next = false;
    return PARSE_INCOMPLETE;
}
```

**详细解析：**

这个解析器处理了嵌入式串口通信的几个关键问题：
1. **帧同步**：0xAA 0x55 作为起始标志
2. **转义机制**：数据中的 0xAA → 0xAA 0x00 防止误识别帧头
3. **状态机**：逐字节处理，不阻塞
4. **校验**：XOR 简单高效
5. **错误恢复**：遇到非法状态自动回到 WAIT_HEAD1 重新同步

实际应用中常见改进：增加超时机制、增加帧序号防重传、CRC 替代 XOR。
