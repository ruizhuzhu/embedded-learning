## 第 1 题答案 — 简答题 — ★★☆

**参考答案：**

**启动流程**：
1. **上电复位**：CPU 从向量表地址 0x00000000（或通过 VTOR 重映射）读取初始 SP 值和 PC
   - 地址 0x00：初始 MSP（主堆栈指针）值
   - 地址 0x04：Reset_Handler 地址 → 赋给 PC
2. **Reset_Handler 执行**：
   a. （可选）禁用全局中断
   b. 将 .data 段从 Flash 复制到 RAM（初始化值在 Flash，运行时要放在 RAM 中才能修改）
   c. 清零 .bss 段
   d. （C++）调用 .init_array 中的全局构造函数
   e. 初始化系统时钟（SystemInit()）
   f. （可选）启用 FPU
   g. 调用 main()

**为什么 .data 要复制到 RAM**：
- Flash 是只读的（或需要特殊操作才能写），而全局变量运行时需要修改
- .data 的初始值在编译时确定，存储在 Flash 中（.rodata 旁边）
- 启动代码将这些初始值复制到 RAM 指定的位置（链接脚本中标定的 .data 段 LMA 和 VMA）

**详细解析：**

理解启动流程对于 debug 启动问题、实现 Bootloader 和自定义链接脚本都是必备知识。

## 第 2 题答案 — 选择题 — ★☆☆

**正确答案：B**

**详细解析：**

MCU 执行 WFI（Wait For Interrupt）或 WFE（Wait For Event）进入休眠。当发生中断时：
1. CPU 唤醒，执行该中断的 ISR
2. ISR 返回后，执行 WFI/WFE 后面的指令继续运行
3. 程序的上下文（寄存器、堆栈）在休眠期间保持不变

不同休眠深度（Sleep、Stop、Standby）行为不同：
- **Sleep**：仅 CPU 时钟停止，外设继续运行，RAM 内容保留
- **Stop**：大部分时钟停止，RAM 保留，通过 EXTI 唤醒
- **Standby**：RAM 丢失，唤醒后等同于复位（需要重新初始化）

## 第 3 题答案 — 编程题 — ★☆☆

**参考实现：**

```c
// 1. 纯位运算版本（总是可用）
uint16_t swap16(uint16_t val) {
    return (uint16_t)((val << 8) | (val >> 8));
}

uint32_t swap32(uint32_t val) {
    return ((val << 24)                  ) |
           ((val <<  8) & 0x00FF0000UL) |
           ((val >>  8) & 0x0000FF00UL) |
           ((val >> 24)                  );
}

// 2. ARM CMSIS 内置高效指令版本
// uint32_t swap32(uint32_t val) {
//     return __REV(val);   // Cortex-M3+ 单周期 REV 指令
// }
// uint16_t swap16(uint16_t val) {
//     return __REV16(val);
// }

// 3. 编译时检测大小端
#if defined(__BYTE_ORDER__) && __BYTE_ORDER__ == __ORDER_LITTLE_ENDIAN__
    #define IS_LITTLE_ENDIAN 1
#elif defined(__BYTE_ORDER__) && __BYTE_ORDER__ == __ORDER_BIG_ENDIAN__
    #define IS_LITTLE_ENDIAN 0
#else
    // 运行时检测（不推荐，编译器可优化掉死代码）
    static inline bool is_little_endian(void) {
        const uint16_t test = 0x0001;
        return (*(const uint8_t*)&test) == 0x01;
    }
#endif

// 4. htonl/ntohl（主机字节序 → 网络字节序 = 大端）
#if IS_LITTLE_ENDIAN
    #define htonl(x) swap32(x)
    #define ntohl(x) swap32(x)
    #define htons(x) swap16(x)
    #define ntohs(x) swap16(x)
#else
    #define htonl(x) (x)
    #define ntohl(x) (x)
    #define htons(x) (x)
    #define ntohs(x) (x)
#endif
```

**详细解析：**

大小端转换在网络通信、跨平台数据交换、文件格式解析中频繁出现。ARM Cortex-M 的 REV 指令可以在单周期内完成 32 位字节反转，效率最高。
