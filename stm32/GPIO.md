### GPIO 通用输入输出接口

    负责外设的控制 数据的输入输出等等作用，MCU中最常用的接口

### 使用流程
    1.打开相关的时钟使能寄存器（RCC_AHBxENRx）
    2.配置端口模式
    3.配置引脚输出类型模式
    4.配置引脚输出速度
    5.配置相关上下拉寄存器
    6.配置引脚输入类型模式
    7.配置输出数据寄存器

### 常见按键消抖方法
    1.延时消抖
    '''if(key_press())
    {
        delay(100);//延时等待抖动过去
        if(key_press)
        {
            //确认按下按键
        }
    }'''
    //优点：简单 易懂
    //缺点：阻塞，占用CPU，不推荐放在主循环中
    2.周期性采样+计数判定 
    #define checktime 5//采样周期，由定时器决定
    int main()
    {
        if(key = key_bouncecheck(key))
        {
            if(key == 0)//确认按下
            {
                //相关操作
            }
        }
    }
    key_bouncecheck()
    {   checked = 4;//四次连续抖动确认为按下按键
        if(key == laststatus)
        {
            count = 0;//与上一次状态一致，清零
        }
        else//发生跳变，不确定抖动还是真按下按键
        {
            count++;
            laststatus = key;更新上一次按键状态
        }
        if(count>=checked)
        {
            return keystatus;//确认按下按键返回按下按键的状态;
        }
    }
    //优点：不阻塞 cpu占用资源少，抗干扰能力强：滤掉随机的毛刺与噪声，相应周期短  4*5ms 几乎无延迟
    3.状态机
    状态转移图
    空闲 IDLE 
           |
           |
           检测到按键按下
           消抖检测
           KEY_DEBOUNCE
           |
           |
           连续低电平
           |
           |
           按键按下
           KEY_PRESSED
           |
           |
           连续高电平
           |
           |
           按键松开(空闲)
           KEY_IDLE
    优点：可以自定义单击，双击，长按按钮的状态，可拓展性强
          状态转移图一目了然，分离了按下，长按，松开的按键状态
    缺点：按键多时，多组自定义按键状态枚举类，且代码过长
    4.软件计时器/RTOS延时任务消抖
    优点：与 RTOS 生态完美融合：利用其任务、队列、定时器等机制，易于与系统其他部分通信。不占用主循环：事件驱动，不阻塞高优先级任务。
    缺点
    依赖 RTOS：必须在 RTOS 环境下。
    开销稍大：创建任务/定时器、上下文切换有额外开销。
    实时性略低：定时器回调执行时间点由系统调度决定，可能有几毫秒抖动。
    5.硬件
    RC+施密特电路消抖
    优点：软件轻松
    缺点：触发时间固定。
